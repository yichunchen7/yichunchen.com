# custom function for model comparison results when using gridsearch to identify high-performing models
def model_comp_results(model_input):
    # Get results from model comparison
    results = pd.DataFrame(model_input.cv_results_, columns=['param_classifier','mean_test_score'])
    
    # Round results and sort in descending order
    results['mean_test_score'] = round(results['mean_test_score'], 3)
    results = results.sort_values(by='mean_test_score', ascending=False)
    return results

Baseline Summary:¶

As we can see, the imbalanced target class requires some refinement in order to hopefully improve model performance. Scoring of the dominant class, False/0, is very good due to how many instances it contains - scoring of the minority class True/1 is poor because there are fewer observations to train the model on.

As a result of this observation, we can look into the use of undersampling and oversampling techniques on the majority/minority class in our target feature. This will hopefully provide a more even ratio between class frequency, and provide a better scoring model.

Resampling Summary We used a researched and proven method of combining SMOTE oversampling over the minority class True/ with an undersample method of RandomUndersampler on the majority class False/0.
We note that our model can now predict class True with much greater accuracy, though this comes at the cost of reduced accuracy in the opposing class.
